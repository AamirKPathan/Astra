# Contributing to ASTRA

Thank you for your interest in contributing to the ASTRA Autonomous Flight System.

Our goal is to build a high-quality engineering project with professional documentation and maintainable software.

---

# Engineering Philosophy

Every contribution should strive to improve:

* Safety
* Reliability
* Readability
* Documentation
* Maintainability

---

# Before You Start

Please:

* Read the Engineering Notebook.
* Check existing Issues.
* Review open Pull Requests.
* Follow the repository coding standards.

---

# Branch Naming

Examples:

```
feature/navigation

feature/guidance

feature/telemetry

feature/ui

fix/logging

docs/notebook

hardware/flight-computer
```

---

# Commit Messages

Use descriptive commit messages.

Examples:

```
feat(nav): implement trajectory estimator

fix(control): correct PID output scaling

docs(EN-0007): update guidance design

hardware(pcb): revise sensor layout

test(sim): add wind model validation
```

---

# Pull Requests

Each Pull Request should include:

* Description
* Motivation
* Engineering rationale
* Testing performed
* Documentation updates
* Related issues

---

# Documentation

All engineering changes must be documented.

Documentation is considered part of the implementation.

---

# Code Standards

* Write readable code.
* Keep functions focused.
* Comment non-obvious logic.
* Remove unused code.
* Prefer modular design.

---

# Testing

New features should include appropriate testing whenever practical.

Simulation is encouraged before hardware implementation.

---

# Questions

Open an Issue if you have questions or would like to discuss a proposed feature before implementation.

Thank you for helping improve ASTRA.
