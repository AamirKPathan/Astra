# Markdown Syntax Rules Table

| Element | Rule | Example |
|--------|------|---------|
| Document Title | Bold, H1 | `# **Title Here**` |
| Section Headers | Bold, H2 | `## **Section Title**` |
| Sub‑Section Headers | Bold, H3 | `### **Subsection Title**` |
| Bold Text | Use `**` | `**important**` |
| Italic Text | Use `*` | `*note*` |
| Code (inline) | Use backticks | `` `code_here` `` |
| Code Block | Use triple backticks | ```code block``` |
| Lists | Hyphens for bullets | `- item` |
| Numbered Lists | Use `1.` `2.` etc. | `1. First` |
| Tables | Use `|` separators | `| A | B |` |
| Horizontal Rule | Use `---` | `---` |
| Links | Standard Markdown | `[text](url)` |
| File/Directory Names | Inline code | `` `README.md` `` |
| Document Metadata | Bold labels + plain text | `**Author:** Name` |
| Quotes | Use `>` | `> quoted text` |
| Emphasis in Headings | Allowed | `## **Important Section**` |
| Acronyms | Define on first use | “ASTRA Autonomous Flight System (AAFS)” |
| Document Footer | Bold or plain | `## **Ad Astra Per Artem**` |
