# ASTRA Versioning Policy

ASTRA uses Semantic Versioning for software and structured revision numbering for engineering documentation and hardware.

---

# Software Versions

Format:

```
MAJOR.MINOR.PATCH
```

Example:

```
1.2.4
```

## MAJOR

Increment when incompatible or significant architectural changes are introduced.

Example:

```
1.0.0 → 2.0.0
```

---

## MINOR

Increment when new functionality is added without breaking compatibility.

Example:

```
1.3.0 → 1.4.0
```

---

## PATCH

Increment for bug fixes, documentation corrections, or minor improvements.

Example:

```
1.4.2 → 1.4.3
```

---

# Engineering Notebook Revisions

Each engineering document maintains its own revision number.

Examples:

```
Rev 0.1
Rev 0.2
Rev 0.3

Rev 1.0
Rev 1.1
Rev 2.0
```

Revision meanings:

* 0.x = Draft
* 1.x = Approved baseline
* 2.x = Major redesign
